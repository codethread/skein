# In-Tree Installer Removal + Test Conversion Proposal

**Document ID:** `PROP-Itr-001` **Last Updated:** 2026-07-24 **Related RFCs:** None **Related ADRs:** [ADR-003](../../adrs/0003-spool-activation-lifecycle.md) (P5 decision C, P6 decision D, P7 conventions, P8 consequences) **Related root specs:** [SPEC-003 repl-api](../../specs/repl-api.md) (P5, P6, C19/C19a), [SPEC-004 daemon-runtime](../../specs/daemon-runtime.md) (C45, C46, C46b, C74a)

## PROP-Itr-001.P1 Problem

Seven in-tree spools (batteries, workflow, executors.shell, guild, chime, cron, unsafe-text-search) still define `install!`, the imperative pre-module activation entry point. The census (epic `waq0l`): 28 call sites, all under `test/`, ~76 fixture executions per suite run, zero production callers. ADR-003.P5 records that deleting them completes PROP-Olr-001.G9/S7 â€” the OLR record already removed the peer installers; the in-tree survivors were a deliberate deferral, not a decision. The core feature (fbr4m, `9cd1345`) shipped the `{:load :image}` grammar and the C46b reconcile contract this conversion consumes, so the tax for converting fixtures to the production activation path is now ~neutral.

The fixture/production split is not just clutter: it concealed the chime engine outage (fixed in `2af05d7`) because tests exercised `install!` while production ran modules. One activation path is the structural fix.

## PROP-Itr-001.P2 Goals

- **PROP-Itr-001.G1:** No `defn install!` remains under `spools/`; every test activates spools via `runtime/module!` with `:load :image` from the `:test` classpath.
- **PROP-Itr-001.G2:** Each spool exports its base module declaration as data (working name `<spool>/module`), the single authored source of the `:ns`/`:contribute`/`:reconcile` triple; tests consume it directly.
- **PROP-Itr-001.G3:** Batteries' reconcile conforms to SPEC-004.C46b: glossary seeding on `:applied` only, no re-registration on `:removed`, loud failure on any other status.
- **PROP-Itr-001.G4:** Guild's optional fallback guild-name survives installer deletion as an explicit exported runtime-state setter with its `guild_test` coverage intact.
- **PROP-Itr-001.G5:** One shared test-support activation helper replaces 28 inline call shapes; shell fixtures activate `:workflow` first with `:after [:skein/spools-workflow]`-style edges (note `4q8cg`).
- **PROP-Itr-001.G6:** Docs converge: 7 regenerated `spools/*.api.md`, spool prose scrubbed of installer-as-current language, and ADR-003.P7's operational conventions moved into `docs/spools/testing.md` (and `writing-shared-spools.md` where they address spool authors), linking to the ADR for rationale (card note `tehh8`).
- **PROP-Itr-001.G7:** A recorded grep-gate allowlist: `rg 'install!' src spools test docs devflow .skein --glob '!devflow/archive/**'` matches only `.skein/kanban_tracker.clj` (workspace config, out of scope), the staged sibling-spool call in `config_ops_test.clj` (converts with the kanban.spool release/cutover), and prose that names `install!` as history.

## PROP-Itr-001.P3 Non-goals

- **PROP-Itr-001.NG1:** Sibling repos (devflow.spool, kanban.spool, agent-harness.spool, dresser.spool) â€” features `9snqu`/`kst0n`. `config_ops_test.clj:155`'s `ct.spools.kanban/install!` resolve stays until the kanban.spool pin bump (feature `9snqu`/`rtnfv` coordination): it can only go green once the consuming pin drops the installer.
- **PROP-Itr-001.NG2:** `.skein/kanban_tracker.clj`'s config-level `install!` â€” a workspace file, not a spool export; epic DONE-WHEN excludes it.
- **PROP-Itr-001.NG3:** Promoting the activation helper into `skein.test.alpha` â€” ADR-003 did not bless promotion; the helper stays in `skein.spools.test-support`. Revisit when sibling conversions (9snqu) show the same shape three times.
- **PROP-Itr-001.NG4:** Deref'ing exported datums from `.skein/init.clj`. Production classpath is `src`+`dev` only â€” spool namespaces are not loadable when startup collection evaluates `init.clj` on a cold start, so the literal declaration maps in `init.clj` stay. The exported datum is the authored source for tests; drift between it and `init.clj` is caught by the suite activating the same triple the datum names. ADR-003.P7's "production assocs `:spools` onto it" applies where the namespace is image-present (sibling-consumer worlds, REPL), not to this repo's cold-start config.
- **PROP-Itr-001.NG5:** Adding removal-path branching to workflow, cron, guild, or unsafe-text-search reconcilers beyond what C46b's defect definition requires. Cron's reconciler is status-agnostic by design (it reconciles against the effective registry, which already reflects removal); workflow and guild run idempotent declarations; none re-registers into a duplicate check on `:removed`. Only batteries has the C46b defect. Shell's silent `{:reconciled :noop}` default branch does get the loud-failure fix â€” it predates C46b and now contradicts it.

## PROP-Itr-001.P4 Proposed scope

- **PROP-Itr-001.S1 (exported datums):** Each of the 7 spools exports `module` â€” e.g. `(def module {:ns 'skein.spools.batteries :contribute 'skein.spools.batteries/contribute :reconcile 'skein.spools.batteries/reconcile})` â€” beside its `contribute`/`reconcile`. Shell's datum carries no `:after` (the module key namespace is workspace-chosen); the fixture supplies the edge. Datums carry no `:spools` and no `:load`: callers assoc what their world needs (ADR-003.P7's two honest variants).
- **PROP-Itr-001.S2 (installer deletion):** Delete all 7 `defn install!` forms. batteries: reconcile branches per C46b â€” `:applied` seeds glossary outcomes, `:removed` is an explicit no-effect branch (the glossary alpha API deliberately has register/replace and no unregister; outcomes are process-lifetime runtime resources), any other status fails loudly (chime's pattern). guild: export `set-fallback-guild-name!` (validates nil-or-non-blank-string loudly, resets the runtime-owned atom, republishes declarations) and convert `guild_test`'s fallback cases to reconcile-then-set. shell: default reconcile branch fails loudly instead of returning silent `:noop`. cron: pure deletion plus the stale ":call at startup" docstring fix. chime, workflow, unsafe-text-search: pure deletion. Docstrings that describe `install!` as current ("legacy direct callers", "the eager entry point", "pre-module lifecycle") rewritten to describe the module lifecycle only.
- **PROP-Itr-001.S3 (test-support helper):** `test-support/activate-spool!` â€” `(activate-spool! rt key base-decl & {:keys [after]})` â€” assocs `:load :image` onto the datum, declares via `runtime/module!` (outside collection this refreshes the module immediately), and fails loudly with the full refresh result unless the module's outcome is applied/unchanged. Fixtures (`with-batteries` and per-ns equivalents) call it; shell fixtures activate `:workflow` then shell with the `:after` edge.
- **PROP-Itr-001.S4 (call-site conversion):** All 28 call sites convert. Tests-of-installer-semantics re-target the module path: `batteries_test` ~1034 asserts glossary seeding via module reconcile; `workflow_test` ~17 asserts `workflow/*` vocab ownership via module reconcile. Comment-only mentions (`vocab_test`, `nvd_scan_test`, `config_test`) reword to name the module lifecycle. Rider R5 audit found no converted test running full `refresh!` (the OLR-style disposable-root sync tests use generated namespaces, untouched); rider R6 holds â€” fixtures activate from the `:test` classpath and approve no real spool roots.
- **PROP-Itr-001.S5 (docs):** `make api-docs` regenerates the 7 `spools/*.api.md`. Prose sweep: `spools/guild.md` ~89, `spools/unsafe-text-search.md` ~57, cookbook mentions (`cron.cookbook.md`, `guild.cookbook.md`, `shell.cookbook.md`, `workflow.cookbook.md`, `batteries.md`, `shell.md`, `workflow.md`). ADR-003.P7's fixture rules and the exported-datum pattern land in `docs/spools/testing.md` + `writing-shared-spools.md` with ADR links (docs-style discipline). SPEC-004.C74a checked for installer-as-current language.
- **PROP-Itr-001.S6 (production config):** `.skein/init.clj` declarations already use `contribute`/`reconcile` and need no shape change (NG4); comments referencing installers update. Canonical world picks the change up via `runtime/refresh!` â€” no weaver restart.
- **PROP-Itr-001.S7 (gates):** Full locked suite at queue acceptance (`flock -w 3600 /tmp/skein-test.lock clojure -M:test`) with the suite delta confirmed ~neutral vs the study-note baselines (image activation); `(cd cli && go test ./...)`; `clojure -M:smoke`; `make fmt-check lint reflect-check docs-check`; grep-gate allowlist recorded in a card note.
