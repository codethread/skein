# Convention-Resolved Spool Declarations (`def spool`) Proposal

**Document ID:** `PROP-Dsp-001` **Last Updated:** 2026-07-24 **Kanban card:** `uwnzl` **Related ADRs:** [ADR-002](../../adrs/0002-no-inline-module-lifecycle-macro.md) (printable-declaration invariant, honored), [ADR-003](../../adrs/0003-spool-activation-lifecycle.md) (supersedes the P7 exported-base-declaration amendment; resolves P9's first revisit trigger; P6 decision D unchanged) **Related root specs:** [SPEC-003 repl-api](../../specs/repl-api.md) (`::module-opts`, C19 — see G9), [SPEC-004 daemon-runtime](../../specs/daemon-runtime.md) (C45/C46/C46b) **Prior art:** [PROP-Itr-001](../rrvnn-intree-installer-removal/proposal.md), [PROP-Sbl-001](../9snqu-siblings-rollout/proposal.md) (epic `waq0l`, installer removal) **Review:** cold-read probe by `sol-high` (run `fgjlu`, 24 findings, all incorporated — see P9)

> **Cold-read note.** This proposal came out of a live design discussion with the user immediately after epic `waq0l` landed. The user's rulings in P2 are **binding direction**, quoted verbatim apart from normalizing the alpha tenet's identifier to its current `TEN-000@1` name, so implementers do not re-derive different guidance from the ADR/tenet record alone. Where this proposal and older P7 prose disagree, this proposal wins; the ADR successor decision (G9) makes that supersession durable.

## PROP-Dsp-001.P1 Problem

Epic `waq0l` removed `install!`, but the replacement still spreads one module's activation contract across three places that must agree by hand:

1. The spool namespace defines `contribute`/`reconcile` fns **and** exports a `(def module {:ns … :contribute … :reconcile …})` datum (7 in-tree copies: `spools/{chime,batteries,unsafe-text-search,workflow,cron,guild}` plus `spools/workflow/src/skein/spools/executors/shell.clj`).
2. Every consumer's `runtime/module!` call copies the `:contribute`/`:reconcile` triple into its opts. Exact conversion inventory in `.skein/init.clj`: **12** ns-backed entries carry entry-point keys (batteries, workflow, shell, unsafe-text-search, devflow, shuttle, delegation, bench, chime, kanban, cron, treadle) and **5** `:file` entries do (`module_adapters.clj` `:reconcile`; `harnesses.clj` both; `reviewers.clj` `:contribute`; `kanban_tracker.clj` `:reconcile`; `workflows.clj` `:contribute`). The 5 `:macros/*` entries and 4 authoring-forms-only files carry none.
3. Cold-start config cannot deref the datum (spool sources are not loadable at collection time), so `.skein/init.clj` mirrors each triple **literally**, policed by a parity test (`test/skein/config_test.clj`, guard-wiring section) — an apparatus the ADR-003.P7 amendment (dated 2026-07-24, the same day as this proposal) itself describes as "recorded, tested duplication, not a second source of truth."

The user's diagnosis, verbatim:

> "I think fail loud has been leant on too heavily. the ergonomics are really very poor and put a lot of burden on users. they have three functions and then a testing layer to understand. We should provide deep-modules and swallow the complexity"

The tenets already license the fix. TEN-003 (FAIL LOUDLY) contains its own escape clause: "If this provides api friction or poor ergonomics, we solve that in other ways like persisted config or **better api design**." TEN-007 names deep-module discipline as the house style; TEN-004 says minimum surface; TEN-000@1 authorizes dropping the old grammar without migration shims (the PROP-Olr-001.G9 precedent: that goal chose outright removal of the old lifecycle over aliases and compatibility shims, and the epic executed it).

## PROP-Dsp-001.P2 Binding direction — user rulings (verbatim)

- **R1 (deep module):** "We should provide deep-modules and swallow the complexity."
- **R2 (target shape):** the user sketched exactly this consumer surface:

  ```clojure
  ;; in the spool namespace — the single authored declaration
  (def spool
    {:contribute ...
     :reconcile ...})

  ;; in consuming config — world policy only, the rest automatic
  (runtime/module! runtime :skein/spools-shuttle
                   {:ns 'ct.spools.agent-run
                    :spools ['ct.spools/agent-run]
                    :required? true})
  ```

- **R3 (convention is API, not a fail-loud violation):** "i just want to cut down on ceremony by introducing 'magic' function names, because whether the namespace declares the correct def name, or the module! call writes the correct keys, they are both a form of contract users read from the docs." And: "we can make the naming convention part of the api." Implementers must not read TEN-003 as forbidding convention-based resolution — the loudness moves inside the deep module (G5), it does not get relaxed.
- **R4 (reload semantics inviolable):** "To be clear, i don't want to break any of runtime/reload semantics we have worked hard to build." See NG1.
- **R5 (breakage policy):** "TEN-000@1 is still valid" — grammar keys are removed, not aliased or shimmed. (The SPEC-003.C19 consequence of this ruling needs explicit treatment: G9 and Q3.)
- **R6 (tests):** "tests absorb the complexity too and have ergonomic handlers for users."
- **R7 (author validation):** "We could likely give users a defprotocol or sdef they can use in tests if they want to be able to validate their `(def module`, although our runtime should make it clear if values are wrong/malformed." (Spec chosen over protocol — NG5.)
- **R8 (`:file` is workable, not a blocker):** "as for module! :file, i think we can resolve that in a better way. Perhaps we dissolve file altogether and force code into .skein/spools or some consistent mechanism, or `:file` just has to workaround this issue in another way, but it all feels workable." (Finding F3 shows `:file` composes with the convention; dissolving `:file` is out of scope — NG2.)
- **R9 (rename):** "i also think we use the opportunity to rename `def module` to `def spool` - less likely to collide and makes intent clearer."

## PROP-Dsp-001.P3 Technical findings (verified against source, 2026-07-24)

- **F1 — failure locality moves in exactly one place.** Declared `:contribute`/`:reconcile` symbols are already resolved at **evaluation time, after source load**, via `resolve-module-fn!` (`src/skein/core/weaver/module_refresh.clj:71-78`, used at `:394-431` and in `reconcile-one` around `:555`), so for ordinary modules convention resolution changes nothing about where failures land. The one exception: image mode's missing-entry-point refusal fires at **declaration** time today (`module_graph.clj:97-102` in `normalize-declaration`; mirrored in the `::module-opts` spec at `src/skein/api/runtime/alpha.clj:211`). G4 deliberately moves that one failure to an evaluation-time `:failed` outcome — the ns cannot be inspected before load/lookup. This is an accepted, named change, not an oversight.
- **F2 — effective entry points are resolved at every module evaluation.** When source is `:unchanged`, an explicit `:contribute` is still called on every refresh (`module_refresh.clj:407-417`); only the no-`:contribute` authoring-forms path reuses the previous contribution. Every effective symbol is therefore resolved and root-value-validated at **every module evaluation**, covering the `:unchanged` path, classpath bindings, targeted refreshes, and namespaces touched by `reload-code!` identically. The `spool` var supplies only fields absent from a Phase A explicit declaration; a complete explicit declaration does not consult the var during the transition. G1 states this normatively.
- **F3 — `:file` modules are ns-scoped, with one new contract clause needed.** Each config file loads into a namespace the coordinator tracks, and `resolve-module-fn!` already resolves unqualified symbols against that loaded ns. But the file→namespace mapping is currently best-effort (a file might declare no `ns`), so G1 adds the clause: a `:file` module using the convention must declare exactly one `ns` form; the coordinator resolves `spool` in that declared namespace. A file with no `ns` declaration keeps today's behavior (authoring forms only, no convention lookup).
- **F4 — image mode improves.** `evaluate-image-module` (`module_refresh.clj:370-392`) requires an explicit `:contribute` today because image mode skips source evaluation, so there is no authoring-forms collection to fall back on. Resolving `<ns>/spool` from the already-loaded image replaces that requirement; a missing `spool` var becomes the loud `:failed` outcome (locality note in F1). This is exactly ADR-003.P9's first revisit trigger ("if explicit-`:contribute` becomes the common friction… revisit whether collection can honestly serve image-owned namespaces"), resolved **without** relaxing any loud failure (the P9 warning, honored — NG4).
- **F5 — the ergonomic test handler already exists.** `activate-spool!` (`test/skein/spools/test_support.clj:161`) wraps datum + `:load :image` + loud outcome assertion. Under the convention it takes a namespace symbol — `(activate-spool! rt :key 'skein.spools.chime)` — and **`require`s that namespace itself** before declaring the image module, so fixtures need no incidental `:require` just to make the image lookup succeed. This satisfies R6.
- **F6 — the target consumer shape already exists and reads well.** The `:macros/*` entries in `.skein/init.clj:85-107` are already `{:ns … :spools … :required? true}` with no entry-point keys.
- **F7 — a real footgun gets fixed, scoped precisely.** Today an explicit `:contribute` silently discards any authoring forms the source load collected (`module_refresh.clj:399-417`). Under the convention, a `spool` var whose `:contribute` is present **and** collected authoring forms in the same load is a loud conflict (G5). The conflict is scoped to `:contribute` only: a **`:reconcile`-only `spool` var composes with authoring forms** — that combination is valid production behavior today (`module_adapters.clj`, `kanban_tracker.clj`: forms-as-contribution plus a reconciler) and must survive the conversion.
- **F8 — no name collisions in this repo's checked scope.** `grep` finds no existing `(def spool` / `(defn spool` in `src/`, `spools/`, or `.skein/` (nearest names: `spool-checkout-root`, `spool-state`, `spool-op`). Scope caveat: this does not survey sibling repos or external consumers; the reservation contract in G5 plus each sibling's own conversion (G10) is what makes the name safe there, not this grep.
- **F9 — empty contributions are a legitimate pattern, not an error.** The `:macros/patterns|ops|queries|rules` modules deliberately load macro-defining namespaces that author no forms ("the macro-defining namespaces carry no authoring forms, so their default contribution is empty" — `.skein/init.clj:83-84`). An ns module with neither a `spool` var nor collected forms must therefore stay legal. Do **not** add an empty-contribution loud failure (an earlier discussion draft suggested one; this finding withdraws it).
- **F10 — stale-var behavior is unchanged from today, by parity.** Clojure's in-place ns reload does not unmap vars whose defining forms were deleted, so a `spool` var removed from source can persist in the loaded ns until a harder reload. This is **not a new gap**: today's symbol resolution has the identical property (a deleted `contribute` fn still resolves from the stale image). The existing pickup ladder (`docs/spools/customisation.md`) is the remedy in both worlds. Implementers must not invent var-cleanup machinery for this; document the parity instead.
- **F11 — removal-by-omission needs retained resolved entry points.** A module removed from a recollected graph is not re-evaluated — its source is never loaded on the removal path — and `reconcile-one` currently recovers the reconciler from the previously stored declaration. Once declarations stop carrying `:reconcile`, the coordinator must retain each module's **last successfully resolved entry-point set** in its state and use it for the `:removed` reconcile (SPEC-004.C46b's teardown obligation). G2a makes this a goal; without it, removal teardown silently disappears — the exact parity-bug class ADR-003.P6 exists to prevent.
- **F12 — ADR-002 settles the datum's value type.** ADR-002 requires declarations to stay printable, diffable data — `:contribute`/`:reconcile` as fully qualified symbols, with fn values "rejected on sight" (ADR-002.O1) because a closure voids `plan`, `status`, and shadow-by-redeclare. `def spool` feeds the same published declaration, so the same invariant applies. Settled in G2 (formerly an open question).
- **F13 — SPEC-003.C19 is implicated and must be addressed, not skipped.** C19 promises accretion-based back-compat within each `skein.api.*.alpha` subnamespace, with breaking rethinks moving to a new subnamespace. Removing accepted `module!` opts keys from `skein.api.runtime.alpha` is a breaking change under C19's own definition. R5 ("TEN-000@1 is still valid") points at recording an explicit exception rather than a new subnamespace, but this supersession must be **written into the SPEC-003 delta and the ADR successor with user sign-off** (G9, Q3) — never landed silently.
- **F14 — pinned sibling suites constrain Phase A.** `make spool-suite-gate` runs the pinned external spool suites against this checkout, and those suites' fixtures declare explicit entry-point keys for **in-tree** namespaces (the probe run reports devflow fixtures activating workflow, kanban fixtures activating batteries/guild, agent-harness fixtures activating batteries). Phase A adds `spool` vars to those same namespaces, so a hard "declared key + `spool` var" conflict would fail the gate before Phase B can land. P7's transitional rule is therefore **precedence, not conflict**: during the window, an explicitly declared key wins over the var, silently and documented as transitional.

## PROP-Dsp-001.P4 Goals

- **PROP-Dsp-001.G1 (the convention):** A module's entry points live in a `(def spool …)` var in its namespace. At every module evaluation the refresh coordinator resolves and root-value-validates every effective entry-point symbol (F2) for `:ns` targets, `:file` targets (via the file's single declared `ns`, F3), and `:load :image` targets (from the JVM image). The convention is documented API: stated once in `skein.api.runtime.alpha`'s docstrings and the spool-authoring guide, enforced loudly by the runtime. During Phase A's transitional window, resolution consults `spool` only for fields absent from the legacy explicit declaration; a complete explicit declaration does not consult or validate the var and remains valid when the target namespace has no `spool` var. Phase C removes the explicit keys and this transitional bypass with them.
- **PROP-Dsp-001.G2 (`::spool` shape — settled):** a map with optional `:contribute` and `:reconcile` **symbols**, at least one present; **no `:ns` key** (the namespace is implicit in where the var lives; the old datum's `:ns` key is dropped in the rename). Fn values are rejected loudly per ADR-002.O1. Unqualified symbols are permitted for ceremony's sake and are **qualified by the coordinator against the declaring namespace at resolution time**, so the published declaration remains fully qualified printable data — ADR-002's invariant holds at the publication boundary. Fully qualified symbols pass through unchanged (and are how one names an entry point in another namespace).
- **PROP-Dsp-001.G2a (resolved-state retention):** the coordinator persists each module's last successfully resolved entry-point set in its runtime state. Removal-by-omission reconciles through that retained set (F11), and `status`/`plan`/refresh results expose the resolved entry points per module (a new resolved field alongside the authored declaration — today's `:modules` shows only the authored graph), including after a failed evaluation (last-good retained, failure reported) and across `reload-code!`.
- **PROP-Dsp-001.G3 (grammar removal):** `::module-opts` drops `:contribute` and `:reconcile` per TEN-000@1 — no aliases, no shims. The consumer surface becomes: source target (`:ns` | `:file`) + world policy (`:spools`, `:after`, `:required?`, `:load :image` — placement settled, formerly open). Removal completes in Phase C (P7); the C19 consequence is G9's.
- **PROP-Dsp-001.G4 (image mode):** delete decision B's explicit-`:contribute` requirement (both the `normalize-declaration` refusal at `module_graph.clj:97-102` and the spec clause at `alpha.clj:211`); image evaluation resolves `<ns>/spool` from the JVM image; a missing var or unloaded ns is a loud evaluation-time `:failed` outcome once no explicit Phase A entry point supplies the field. The declaration→evaluation locality move for this one case is accepted and named (F1).
- **PROP-Dsp-001.G5 (loud enforcement, enumerated):** evaluation-time failures with named-field, actionable errors for: a consulted `spool` var value that is not a map, has unknown keys, or has neither entry point; an entry-point value that is not a symbol (ADR-002.O1); a symbol that does not resolve, or resolves to a var whose **root value** is not a fn — the check derefs and tests the value, since a clojure Var is itself `ifn?` regardless of what it holds, so `(ifn? var)` cannot give the promised error; a `spool` var carrying `:contribute` while the same load collected authoring forms (loud conflict forcing the author to choose — F7; `:reconcile`-only vars compose with forms and do not conflict); an image module whose ns lacks a `spool` var once no explicit Phase A entry point supplies contribution (F4). The reservation contract: outside Phase A's complete-explicit bypass, a **public** var named `spool` in a module's namespace _is_ the module declaration; private vars are ignored, namespaces never activated as modules are unaffected, and Q2's lint proposal is the guard against incidental shadowing inside this repo. Phase C removes the bypass, making the reservation unconditional. Empty contribution without a `spool` var stays legal (F9). Stale vars after in-place reload keep today's semantics (F10).
- **PROP-Dsp-001.G6 (author validation surface, single-source):** register `s/def ::spool` in `skein.api.spool.alpha`, and implement the runtime's own G5 validation **over that same spec** so the convenience surface and the enforcement path cannot drift. Authors validate with plain `s/valid?`/`s/explain-data` against `::spool`; no separate helper fn ships unless the implementer finds the spec alone insufficient for actionable errors, in which case the helper is a thin named wrapper over the same spec, not a second validator.
- **PROP-Dsp-001.G6a (repository lint — settled):** `lint-conventions` rejects a public `spool` var in this repository's module-loadable namespaces when its authored value does not satisfy `::spool`. The rule uses the existing scan and aggregation seams in `scripts/quality/conventions_check.clj`, with ratchet coverage in `test/skein/quality/conventions_check_test.clj`. It is a repository guard, not a second runtime contract and not protection for external consumers; runtime validation remains authoritative everywhere.
- **PROP-Dsp-001.G7 (in-tree conversion):** rename the 7 in-tree `def module` datums to `def spool` and drop their `:ns` keys; the 5 entry-point-carrying file modules (P1 inventory) gain `(def spool …)` in their file ns; every `.skein/init.clj` entry drops `:contribute`/`:reconcile` (in-tree entries in Phase A, sibling-backed entries in Phase C); the literal-mirror pattern and parity test are retired per the P7 schedule. Epic DONE-WHEN gates (they hold at **Phase C completion**, not at Phase A): `git grep '(def module'` in `spools/` returns nothing, and `grep ':contribute\|:reconcile' .skein/init.clj` returns nothing.
- **PROP-Dsp-001.G8 (test ergonomics):** `activate-spool!` takes a namespace symbol and requires it itself (F5); fixtures stop importing spool namespaces just to reach a datum; `docs/spools/testing.md`'s exported-base-declaration section collapses to the one-line convention.
- **PROP-Dsp-001.G9 (record):** an ADR-003 successor decision (or new ADR) superseding the P7 exported-base-declaration amendment and recording P9-trigger resolution; deltas to SPEC-003 (`::module-opts`, `::spool`, and an **explicit C19 supersession record** for the `module!` key removal — the accretion promise is knowingly broken under TEN-000@1 and R5, with the new-subnamespace alternative considered and rejected as shim-shaped; needs user sign-off, Q3) and SPEC-004 (C45/C46 image mode, C46b retained-entry-point teardown); prose updates in `docs/spools/testing.md`, `docs/spools/writing-shared-spools.md`, `docs/spools/customisation.md`; `make api-docs` regeneration. **Spec truthfulness per phase:** each phase's land updates the root specs to describe that phase's actual behavior — Phase A's delta documents the convention _and_ the transitional acceptance of the old keys as a recorded pending-removal obligation; Phase C's delta completes the removal. At no land is a root spec knowingly false.
- **PROP-Dsp-001.G10 (siblings + cutover):** devflow.spool, kanban.spool, and agent-harness.spool: export `(def spool …)`, delete the `module` export, **and convert their own consuming surfaces in the same release** — each repo's own workspace config, test fixtures, activation helpers, and docs stop declaring explicit entry-point keys for any namespace that now carries a `spool` var (F14's precedence rule keeps the old shape working during the window, but the siblings' own releases ship converted). Each convention-dependent release **raises its `:skein/min` floor to `v1`**, which must include landed Phase A, so an older runtime cannot select it. The separate hitl card `b3v1r` owns that Skein stamp; this feature must not create it, and Phase B remains blocked until `b3v1r` closes with the verified marker. New `v<int>` sibling markers carry `release-exception.md` records per each repo's precedent (removing an exported name is a break; breaks take new names/markers, never aliases). The cutover feature bumps pins, converts the sibling-backed init.clj entries, removes the grammar keys (Phase C), refreshes the canonical world, and verifies live — the same close-out shape feature `rtnfv` performed for epic `waq0l` (pin bumps + canonical-world refresh + live chime verification). Sibling suites hard-code `../skein-src` (recorded in `waq0l` note `5bae1`), so Phase A lands before the external stamp and sibling releases.

## PROP-Dsp-001.P5 Non-goals

- **PROP-Dsp-001.NG1 (reload semantics untouched — R4):** staging vs. refresh collection, full-refresh recollection from startup files, source stamps and the `:unchanged` fast path, residual/conflict/remedy reporting, root guards (`module-root-problem`), dependency ordering, and the reconcile status-branching contract (ADR-003.P6 decision D / SPEC-004.C46b) are all unchanged. Only _where the entry-point triple comes from_ changes — plus the retention seam G2a adds so removal teardown keeps working.
- **PROP-Dsp-001.NG2 (`:file` not dissolved):** F3 shows `:file` composes with the convention (with the single-`ns` clause). The user's alternative ("dissolve file altogether and force code into .skein/spools or some consistent mechanism" — R8) is recorded as a possible future card, not scope here.
- **PROP-Dsp-001.NG3 (no kernel teardown callbacks):** the CC13 boundary stands — DELTA-OlrRepl-001.CC13 keeps generic lifecycle/teardown callbacks out of the publication kernel; teardown remains the reconcile fn's `:removed` branch. Nothing here adds hooks.
- **PROP-Dsp-001.NG4 (no loudness relaxation):** every refusal the deleted grammar clauses performed must reappear as a loud failure (G5) — evaluation-time where locality moves (F1) — never a silent default. ADR-003.P9: "Do not revisit by relaxing the loud failures."
- **PROP-Dsp-001.NG5 (no defprotocol):** R7 offered protocol-or-spec; spec wins. A protocol forces a type where a map suffices and adds nothing the runtime enforcement (G5) does not already give.
- **PROP-Dsp-001.NG6 (no var-cleanup machinery):** stale-var behavior after in-place reload keeps exact parity with today's symbol resolution (F10); the pickup ladder is the documented remedy.

## PROP-Dsp-001.P6 Open questions

- **PROP-Dsp-001.Q1 (settled — recorded for history):** symbols vs. fn values was open in the first draft; ADR-002.O1 settles it (F12, G2). Fn values are rejected; unqualified symbols are coordinator-qualified.
- **PROP-Dsp-001.Q2 (settled — reserve the name):** the user approved the repository-only `lint-conventions` guard in G6a. It catches an incidental or malformed public `(def spool …)` in module-loadable namespaces before merge. The runtime contract remains G5's public-var rule; the lint cannot protect external consumers (F8).
- **PROP-Dsp-001.Q3 (C19 supersession wording — needs user sign-off):** G9 requires the SPEC-003 delta to record the deliberate break of C19's accretion promise for `skein.api.runtime.alpha/module!`. The recommended shape: a one-paragraph C19 exception citing TEN-000@1 and ruling R5, noting the new-subnamespace alternative was considered and rejected (a parallel `module!` under a new name is exactly the alias/shim R5 forbids). The user reviews that wording before it lands; implementers do not land it silently.
- **PROP-Dsp-001.Q4 (file-module naming wrinkle):** workspace `:file` modules (`kanban_tracker.clj` etc.) are not spools, yet under one uniform convention they too declare `(def spool …)`. Uniformity (one name, one rule) is recommended over a second var name for files; record the misnomer consciously rather than fork the convention.

## PROP-Dsp-001.P7 Landing order and the transitional grammar window

`.skein/init.clj` must stay valid against **pinned** sibling releases at every land, and `make spool-suite-gate` runs pinned sibling suites whose fixtures still declare explicit entry-point keys for in-tree namespaces (F14). That makes G3's key removal the _last_ step, and it fixes the transitional rule's shape:

1. **Phase A (skein-src, core + in-tree):** coordinator resolves `spool` per G1/G2/G2a; grammar _temporarily_ still accepts `:contribute`/`:reconcile`, and during the window an **explicitly declared key takes precedence over the `spool` var per key** — silently, documented in the Phase A spec delta as transitional. The coordinator consults the var only for absent fields; a complete legacy declaration does not consult or validate it and remains valid when its target has no `spool` var. A hard conflict here would fail the pinned sibling suites the moment in-tree namespaces gain `spool` vars (F14). The G5 forms-vs-`:contribute` conflict is unaffected (it involves the var and collected forms, not declared keys). In-tree spools rename to `def spool`; in-tree init.clj entries drop their triples. The parity test is **narrowed, not deleted**: it keeps asserting the remaining sibling-backed literal triples against the sibling exports while both exist (precedence hides var/key drift, so mirroring still needs its guard), and dies in Phase C with the literals it polices.
2. **Phase B (siblings, after external Skein v1 stamp):** after Phase A lands and hitl card `b3v1r` closes with the Skein `v1` marker, devflow/kanban/agent-harness export `spool`, delete `module`, convert their own fixtures/configs/helpers/docs, raise `:skein/min` floors to `v1`, and release new markers per G10. The coordinator may prepare this work earlier, but must not publish convention-dependent releases against an unmarked core.
3. **Phase C (cutover):** bump pins, drop the remaining sibling triples from init.clj, convert the remaining core tests and fixtures that exercise or generate explicit entry-point keys — known inventory to sweep: `test/skein/weaver_test.clj`, `test/skein/api/runtime/alpha_test.clj`, `test/skein/config_ops_test.clj`, and any generated startup-config fixtures (grep for `:contribute`/`:reconcile` under `test/`) — then **remove `:contribute`/`:reconcile` from `::module-opts` entirely** (completing G3), delete the narrowed parity test, land the Phase C spec delta, refresh the canonical world, verify live. The G7 grep gates hold from here.

The window is a landing-order necessity inside one epic, not a compatibility shim for external consumers — it closes in Phase C, and no release ships with the window open as its end state. TEN-000@1 covers the break; sibling `:skein/min` floors gate their consumers.

## PROP-Dsp-001.P8 Suggested slicing

- **S1 (core):** coordinator resolution (G1/G2) + resolved-state retention and status/plan exposure (G2a) + Phase A transitional precedence + G5 loud paths + `::spool` spec in `skein.api.spool.alpha` (G6) + repository lint and ratchet tests (G6a) + Phase A SPEC deltas (G9, including the Q3 C19 wording for user review) + coordinator tests. The coordinator gates include:
  - a precedence-window matrix proving explicit values win per key, absent fields come from `spool`, and complete legacy explicit declarations still work with no `spool` var;
  - a true `:load :image` path with only a preloaded namespace as setup, no source collection/load and no injected callable, covering both success and missing-`spool` failure;
