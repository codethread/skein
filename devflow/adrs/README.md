# ADRs

Architecture Decision Records. An ADR captures a load-bearing decision and, more importantly, the reasoning behind it, so the choice is repeatable and a returning reader learns *why* rather than re-deriving it.

An ADR differs from an [RFC](../rfcs/): an RFC proposes work that then ships; an ADR records a decision reached — including a decision to *not* build something and keep an existing [tenet](../TENETS.md). ADRs may be written retrospectively.

Numbered `NNNN-slug.md`, oldest first. Status is one of `Accepted`, `Superseded`, `Rejected`.
