**Document ID:** `PLAN-LargeAttrScaling-001` **Feature:** `large-attr-scaling` **Proposal:** [proposal.md](./proposal.md) **RFC:** None (no RFC covers attribute storage representation; adjacent [RFC-002 Task Query DSL](../../rfcs/2026-06-24-task-query-dsl.md) defines the query language the assessed read paths run over, unchanged here) **Root specs:** [Strand Model](../../specs/strand-model.md), [Weaver Runtime](../../specs/daemon-runtime.md), [Alpha Surface](../../specs/alpha-surface.md) — context only; this spike stages **no** spec deltas (`PROP-LargeAttrScaling-001.NG2`) **Feature specs:** none **Builds on:** the shipped [eav-attr-storage](../26-07-07__eav-attr-storage/eav-attr-storage.plan.md) (`EAS-PLAN-001`) baseline — EAV row storage, the `archived` cold tier + `WHERE archived = 0` partial indexes, and the merge-blocking benchmark gate `EAS-PLAN-001.P7`/`BG1`–`BG4` — and [attr-scaling-ship-now](../26-07-06__attr-scaling-ship-now/attr-scaling-ship-now.plan.md) (`ASSN-PLAN-001`, L0a pragmas + L1 lean reads). **Status:** Complete **Last Updated:** 2026-07-23

## PLAN-LargeAttrScaling-001.P1 Goal and scope

Turn the `PROP-LargeAttrScaling-001` assessment spike into buildable slices. It delivers three things and no more: a **committed, reusable synthetic large-attr load harness** (`S1`), a **run of that harness on the shipped storage baseline with a committed numbers report** (`S2`), and an **evidence-backed options verdict** for the residual read-path costs the EAV design accepted-but-never-measured (`S3`). Everything the harness measures runs against the storage the shipped `eav-attr-storage` feature already left in place; this spike changes no storage code, stages no spec delta, and adds no public CLI or agent surface (`PROP-LargeAttrScaling-001.NG1`–`NG3`). Any storage change the verdict recommends is a future feature with its own proposal — not built here.

## PLAN-LargeAttrScaling-001.P2 Approach

- **PLAN-LargeAttrScaling-001.A1 (reconcile the existing harness, do not duplicate it):** `dev/skein/eav_benchmark.clj` (`ns skein.eav-benchmark`, committed on `main` at `3d1a99e` as the `eav-attr-storage` merge gate) already reproduces `EAS-PLAN-001.BG1`–`BG3` on paired synthetic fixtures in hand-SQL; `BG4` is the external `flock -w 3600 /tmp/skein-bench.lock` serialization discipline (`eav-attr-storage.plan.md:101`), not something the namespace itself enforces, and stays a plan-level invocation requirement (`V3`, `T2`). It is **not** the tool `PROP-LargeAttrScaling-001.F1` calls missing: it hand-writes both schemas' SQL to prove the document→EAV *migration ratio*, it exercises no shipped `skein.core.*` code, and it covers none of the `F2` large-value / archived-row regimes. The plan **generalizes it into the durable baseline** rather than growing a second harness — its proven synthetic-fixture and write-amp methodology is lifted intact so the accepted gate numbers stay re-runnable, and the shipped-code + `F2` measurement is built on top.
- **PLAN-LargeAttrScaling-001.A2 (two measurement families, one harness):** The harness carries two families that answer two different questions. **(a) Gate reproduction** (`BG1`–`BG4`) is inherently a document-vs-EAV comparison, and the document column no longer exists in shipped code, so it keeps the paired synthetic-fixture / hand-SQL methodology already in `eav_benchmark.clj`. **(b) Shipped-baseline residual paths** (`F2`) measures the *absolute* cost of the real read paths — full-fidelity point read (`strand-row-by-id`, archived included), lean list/`ready` assembly (`attribute-json-by-strand-id`), and the text-search `LIKE` spool — through the actual `skein.core.db` / `skein.spools.text-search` code, on a disposable weaver world, under the `F2` regimes (values around the 1024-byte lean floor, inlined payloads to MB scale, and populated archived-row volumes).
- **PLAN-LargeAttrScaling-001.A3 (real code, real runtime, no published singleton):** Family (b) boots a disposable world from an in-process temp dir with `:publish? false` and runs every measurement under `skein.core.weaver.runtime/with-runtime-binding`, passing the runtime explicitly — it never touches the ambient published runtime and never the canonical `.skein` world. It seeds attributes through the real write path so the rows, indexes, and `archived` flags are exactly what production carries. Family (a) builds throwaway SQLite files in a temp dir and needs no runtime.
- **PLAN-LargeAttrScaling-001.A4 (home and shape — resolves `Q1`):** The harness is committed in the **`test/` tier** (it needs the `text-search` spool and `skein.core.*`/`skein.api.*` on the classpath, which the `:test` alias already provides) and exposes **both** shapes the open question named: a **fast structural smoke** `deftest` at tiny `N` that the per-slice cold gate (`clojure -M:test`) always runs — deterministic, wiring-only, **no** wall-clock assertion — and a **slow full-scale run** that is a plain `-main`/function (never a `deftest`, so the default suite never triggers it), gated by an env flag and serialized behind the benchmark lock. This is the "both" of `PROP-LargeAttrScaling-001.Q1`.
- **PLAN-LargeAttrScaling-001.A5 (informational, not gating — resolves `Q3`):** Because the spike ships no storage change (`NG2`), **no** large-value scenario becomes merge-blocking. The only blocking test is the structural smoke (the harness compiles and every scenario is wired to the real code path at tiny `N`); every timing number is informational and lives in the committed report. The reproduced `BG1`–`BG4` targets are preserved as the recorded durable baseline for a future storage change to gate against — they are not re-imposed as a gate here.
- **PLAN-LargeAttrScaling-001.A6 (pin the baseline — resolves `Q2`):** The report records the exact `main` commit the `S2` run measured and the full documented synthetic-world seed (the `eav_benchmark.clj` `default-options` profile — `seed 1337`, `N 250000`, base attrs + `body` payload every 50th row — extended with the `F2` knobs: a value bucket straddling the 1024-byte lean floor, an MB-scale payload bucket, an archived-row fraction, and a text-search corpus). A later re-run on a new `main` is comparable by construction.
- **PLAN-LargeAttrScaling-001.A7 (options bounded by the settled storage — feeds `S3`/`Q4`):** The residual-options synthesis reads only the `S2` numbers and proposes only mitigations that stay inside EAV+archive and TEN-007 (no option may make a consumer above `skein.core.*` depend on attribute-storage shape). It notes that the assembly-path lever `EAS-DELTA-001.Q1` named — the `(strand_id) WHERE archived = 0` partial index (`idx_attributes_strand_hot`) — already ships, so the assembly question is "is the shipped index enough," not "add one." It ends in one concrete verdict: a named follow-up feature or an explicit no-action, with `Q4` (text-search seeing archived rows: correctness concern vs. pure cost) decided from the measured `LIKE` numbers, not asserted.

## PLAN-LargeAttrScaling-001.P7 Risks and open questions (Q1–Q4 resolved)

- **PLAN-LargeAttrScaling-001.R1 (harness measures a fiction):** a hand-SQL harness that never touches shipped code can report numbers that do not reflect production. Mitigation: the `F2` family measures through the real `skein.core.db` / `text-search` code on a real (disposable) runtime (`A3`); the smoke `deftest` fails loudly if a scenario stops calling the shipped path.
- **PLAN-LargeAttrScaling-001.R2 (slow run leaks into the default suite):** a heavy 250k run wired as a `deftest` would blow every `clojure -M:test`. Mitigation: the full-scale run is a plain env-gated `-main`/fn, never a `deftest`; only the tiny structural smoke is a test (`A4`).
- **PLAN-LargeAttrScaling-001.R3 (bench scratch pollutes the tree):** real SQLite fixtures spawn `-wal`/`-shm` sidecars and multi-GB files. Mitigation: scratch stays under a gitignored `target/`; only the small result EDN and the report are committed; `V7` enforces the clean tree.
- **PLAN-LargeAttrScaling-001.R4 (published-singleton reach):** grabbing the ambient runtime instead of the disposable one would couple the harness to a live weaver. Mitigation: `:publish? false` + `with-runtime-binding` with the runtime passed explicitly, per repo runtime-publication discipline (`A3`).
- **PLAN-LargeAttrScaling-001.R5 (scope creep into a storage change):** the assessment tempts a "quick fix" to a residual path. Mitigation: `NG2` — no `skein.core.*` edit in this spike; `S3` recommends, it does not build (`A7`).
- **PLAN-LargeAttrScaling-001.Q1 — resolved:** harness home/shape = **both** — a committed `test/`-tier harness with a fast structural smoke slice for the cold gate plus a slow full-scale run gated by an env flag and the bench lock (`A4`).
- **PLAN-LargeAttrScaling-001.Q2 — resolved:** the baseline pins to the exact `main` sha the `S2` run measured plus the documented seed profile (`A6`), both recorded in the report.
- **PLAN-LargeAttrScaling-001.Q3 — resolved:** gates stay **informational** (`A5`); no large-value scenario is merge-blocking, since the spike ships no storage change. A future storage feature owns any new gate.
- **PLAN-LargeAttrScaling-001.Q4 — deferred to measurement:** whether text-search seeing archived rows is a correctness concern or purely a cost question is decided in `S3` from the measured `LIKE`-over-archived numbers (`A7`); the plan sets up the measurement rather than pre-deciding.

### PLAN-LargeAttrScaling-001.DN2 `deps.edn`/usage-string ownership deviated from T1 to T4 — 2026-07-11

- T1 shipped `test/skein/large_attr_benchmark.clj` with a usage string documenting `clojure -M:test -m skein.large-attr-benchmark ...` as the invocation, touching no `deps.edn` alias. T2's full-scale run discovered that form does not work: the `:test` alias pins `:main-opts` to `["-m" "skein.test-runner"]`, and `clojure -M` concatenates alias `:main-opts` with any CLI-supplied `-m`/args rather than letting the CLI `-m` replace it, so the documented invocation resolves to `skein.test-runner` choking on the trailing args. T2 worked around this without patching (per its `OS3`/`NG2` boundary) and flagged it to the coordinator. T4 then fixed it by adding the `:large-attr-bench` alias to `deps.edn` and correcting the harness's usage string — a `deps.edn` touch T4's task file did not originally scope, but which is the natural place for a wrap-up slice to land a one-line invocation fix discovered mid-feature. The `P8` table above and `tasks/001-harness.md`/`tasks/004-wrapup-gates.md` are updated to reflect this actual ownership rather than the originally-scoped split.
