# Go CLI JSON Socket Protocol

**Document ID:** `GOCLI-PROTO-001` **Feature:** [Go CLI Migration](../proposal.md) **Status:** Implemented **Last Updated:** 2026-06-25 **Related specs:** [`SPEC-004-D002`](./daemon-runtime.delta.md), [`SPEC-002-D003`](./cli.delta.md)

## GOCLI-PROTO-001.P1 Purpose

This note freezes the minimal cross-language JSON contract between the Go `todo` CLI and the Clojure daemon for this feature. The daemon owns all storage, query registry, and domain semantics; the Go CLI parses command flags, discovers a matching daemon, sends allowlisted JSON requests over a local Unix socket, and formats JSON or human output.

## GOCLI-PROTO-001.P2 Runtime metadata

### GOCLI-PROTO-001.M1 File strategy

- **GOCLI-PROTO-001.M1.1:** Runtime discovery remains keyed by the canonical database path.
- **GOCLI-PROTO-001.M1.2:** The daemon writes Go-readable JSON metadata atomically under the deterministic runtime metadata directory for that canonical database path.
- **GOCLI-PROTO-001.M1.3:** The deterministic runtime metadata directory is the existing daemon runtime directory: `${java.io.tmpdir}/todo-daemon` for the Clojure daemon implementation. The database key is the lowercase hex SHA-256 digest of the canonical database path encoded as UTF-8.
- **GOCLI-PROTO-001.M1.4:** The Go-readable metadata file path is `${runtime_dir}/${database_key}.json`. During transition, existing Clojure nREPL metadata may continue to use `${runtime_dir}/${database_key}.edn`; that EDN file is for Clojure clients only and Go clients must not parse it.
- **GOCLI-PROTO-001.M1.5:** The daemon socket path is advertised by JSON metadata. The implementation uses a shortened stable hash filename under `${runtime_dir}` to stay below Unix socket path length limits.
- **GOCLI-PROTO-001.M1.6:** Missing, malformed, stale, mismatched, or unreachable metadata is an error. Clients fail loudly and must not open SQLite directly.

### GOCLI-PROTO-001.M2 JSON shape

```json
{
  "protocol_version": 1,
  "pid": 12345,
  "database_path": "/absolute/canonical/todo.sqlite",
  "daemon_id": "2f5fd2c8-4d1f-4bc4-8d6c-f7d8ac16a245",
  "socket_path": "/tmp/todo-daemon/7d0f0a0f4b2d4e6c.sock",
  "started_at": "2026-06-25T12:00:00Z",
  "nrepl": {
    "host": "127.0.0.1",
    "port": 55555
  }
}
```

- **GOCLI-PROTO-001.M2.1:** `protocol_version` is the integer metadata/protocol major version. This feature defines version `1` only.
- **GOCLI-PROTO-001.M2.2:** `pid` is the daemon OS process id.
- **GOCLI-PROTO-001.M2.3:** `database_path` is the canonical absolute path for the daemon-owned SQLite database.
- **GOCLI-PROTO-001.M2.4:** `daemon_id` is the daemon identity/nonce generated for this daemon lifetime. Clients echo it in every request.
- **GOCLI-PROTO-001.M2.5:** `socket_path` is the local Unix domain socket path for JSON requests.
- **GOCLI-PROTO-001.M2.6:** `started_at` is an RFC 3339 UTC timestamp for display/debugging.
- **GOCLI-PROTO-001.M2.7:** `nrepl` retains Clojure client endpoint data. It may be `null` only if a later spec removes nREPL publication; Go clients ignore it.

## GOCLI-PROTO-001.P3 Connection framing and timeouts

- **GOCLI-PROTO-001.T1:** Transport is a local Unix domain stream socket. Remote TCP access, authentication, and multi-user authorization are outside this feature.
- **GOCLI-PROTO-001.T2:** Each connection carries exactly one request and one response. The client writes one UTF-8 JSON object followed by `\n`, half-closes or flushes the write side, then reads one UTF-8 JSON object followed by `\n`.
- **GOCLI-PROTO-001.T3:** The server closes the connection after writing the response. Clients must send only one request per connection; extra client data is unsupported and may be ignored when the server closes the connection.
- **GOCLI-PROTO-001.T4:** Default client dial timeout is 1 second. Default request/response deadline is 10 seconds. `stop` uses the same request deadline for acknowledgement; daemon process exit may complete after the response.
- **GOCLI-PROTO-001.T5:** Dial timeout, socket I/O timeout, malformed JSON, unknown protocol version, unknown operation, and identity mismatch are transport/protocol errors and produce non-zero CLI exits.

## GOCLI-PROTO-001.P4 Request envelope

```json
{
  "protocol_version": 1,
  "request_id": "018ff8a0-2f6b-7bb2-8b03-ea6f0ad6f4bb",
  "daemon_id": "2f5fd2c8-4d1f-4bc4-8d6c-f7d8ac16a245",
  "database_path": "/absolute/canonical/todo.sqlite",
  "operation": "show",
  "arguments": {
    "id": "task-1"
  },
  "options": {
    "format": "json"
  }
}
```

- **GOCLI-PROTO-001.REQ1:** `protocol_version` is required and must be `1`.
- **GOCLI-PROTO-001.REQ2:** `request_id` is required, generated by the client, and echoed in the response for diagnostics.
- **GOCLI-PROTO-001.REQ3:** `daemon_id` is required and must match metadata for the selected daemon lifetime.
- **GOCLI-PROTO-001.REQ4:** `database_path` is required and must equal the daemon canonical database path.
- **GOCLI-PROTO-001.REQ5:** `operation` is required and must be in the allowlist in `GOCLI-PROTO-001.P6`.
- **GOCLI-PROTO-001.REQ6:** `arguments` is a JSON object containing operation-specific data. Use `{}` when the operation has no arguments.
- **GOCLI-PROTO-001.REQ7:** `options` is a JSON object for output-relevant or cross-cutting options. This feature defines only `format`, whose values are `human` or `json`. The daemon may ignore `format` for semantic execution but may use it for status payload hints.
- **GOCLI-PROTO-001.REQ8:** Missing required fields, unknown top-level fields, unknown option fields, or wrong JSON types are malformed requests and fail loudly.

## GOCLI-PROTO-001.P5 Response envelope and errors

### GOCLI-PROTO-001.R1 Success response

```json
{
  "protocol_version": 1,
  "request_id": "018ff8a0-2f6b-7bb2-8b03-ea6f0ad6f4bb",
  "ok": true,
  "result": {
    "id": "task-1",
    "title": "Write docs",
    "status": "todo",
    "attributes": {}
  },
  "error": null
}
```

- **GOCLI-PROTO-001.R1.1:** `ok: true` means `result` contains the operation result and `error` is `null`.
- **GOCLI-PROTO-001.R1.2:** Task rows in `result` use JSON objects/arrays with normalized `attributes` rather than JSON-encoded strings.
- **GOCLI-PROTO-001.R1.3:** `status` result is `{ "healthy": true, "pid": integer, "database_path": string, "daemon_id": string, "socket_path": string, "nrepl": object|null }`. `healthy` is true only after metadata validation and socket identity check pass.
- **GOCLI-PROTO-001.R1.4:** `stop` result is `{ "stopping": true, "pid": integer, "database_path": string, "daemon_id": string }`.

### GOCLI-PROTO-001.R2 Error response

```json
{
  "protocol_version": 1,
  "request_id": "018ff8a0-2f6b-7bb2-8b03-ea6f0ad6f4bb",
  "ok": false,
  "result": null,
  "error": {
    "type": "domain",
    "code": "task/not-found",
    "message": "Task not found: missing-task",
    "details": {
      "id": "missing-task"
    }
  }
}
```

- **GOCLI-PROTO-001.R2.1:** `ok: false` means `result` is `null` and `error` is present.
- **GOCLI-PROTO-001.R2.2:** `error.type` is one of `domain`, `protocol`, or `transport`.
- **GOCLI-PROTO-001.R2.3:** `error.code` is a stable slash-delimited string suitable for tests and scripting.
- **GOCLI-PROTO-001.R2.4:** `error.message` is a useful human-readable message.
- **GOCLI-PROTO-001.R2.5:** `error.details` is a JSON object with structured context. Use `{}` when no context applies.
- **GOCLI-PROTO-001.R2.6:** Domain errors are daemon semantic failures such as missing task, invalid status, invalid edge, missing named query, or missing query parameter.
- **GOCLI-PROTO-001.R2.7:** Protocol errors are malformed requests, unsupported protocol versions, disallowed operations, or identity/database mismatches.
- **GOCLI-PROTO-001.R2.8:** Transport errors are socket-level failures observed by the server after a request is accepted. Client-side dial/read/write failures do not have a daemon response envelope but must be surfaced with equivalent non-zero CLI behavior.

## GOCLI-PROTO-001.P6 Operation allowlist

- **GOCLI-PROTO-001.OP1:** The JSON socket transport exposes only these operations for this feature: `init`, `add`, `update`, `show`, `list`, `ready`, `list-query`, `ready-query`, `status`, and `stop`.
- **GOCLI-PROTO-001.OP2:** `register-query`, `load-queries`, `queries`, and `resolve-query` are explicitly excluded from the low-privilege JSON socket transport.
- **GOCLI-PROTO-001.OP3:** Query definition, loading, registry inspection, and rich query debugging remain Clojure REPL/trusted daemon config workflows.

## GOCLI-PROTO-001.P7 Operation arguments

- **GOCLI-PROTO-001.ARG1:** `init`: `{}`.
- **GOCLI-PROTO-001.ARG2:** `add`: `{ "title": string, "status": "todo"|"done"|"failed"|"cancelled", "attributes": object }`. `title`, `status`, and `attributes` are required on the wire. The Go CLI applies the public CLI default status of `todo` before sending the request when the user omits `--status`. `attributes` is `{}` when no attributes are supplied.
- **GOCLI-PROTO-001.ARG3:** `update`: `{ "id": string, "title": string|null, "status": "todo"|"done"|"failed"|"cancelled"|null, "attributes": object|null, "edges": [{ "type": string, "to": string }] }`. All listed keys are required on the wire. `title: null` and `status: null` mean no change. `attributes: null` means no attribute patch; an object, including `{}`, is an attribute patch map. `edges: []` means no edge writes; edge deletion is not part of this feature.
- **GOCLI-PROTO-001.ARG4:** `show`: `{ "id": string }`.
- **GOCLI-PROTO-001.ARG5:** `list`: `{}`.
- **GOCLI-PROTO-001.ARG6:** `ready`: `{}`.
- **GOCLI-PROTO-001.ARG7:** `list-query`: `{ "query": string, "params": object }`, where `params` values are strings.
- **GOCLI-PROTO-001.ARG8:** `ready-query`: `{ "query": string, "params": object }`, where `params` values are strings.
- **GOCLI-PROTO-001.ARG9:** `status`: `{}`.
- **GOCLI-PROTO-001.ARG10:** `stop`: `{}`.

## GOCLI-PROTO-001.P8 Lifecycle semantics

- **GOCLI-PROTO-001.L1:** `daemon status` first reads JSON runtime metadata for the selected canonical database, validates required metadata fields, then connects to `socket_path` and sends a `status` RPC echoing `daemon_id` and `database_path`.
- **GOCLI-PROTO-001.L2:** `daemon status` reports metadata plus socket health/identity. A pid alone is not sufficient for healthy status.
- **GOCLI-PROTO-001.L3:** `daemon stop` first performs the same metadata and identity validation, then sends a `stop` RPC over the socket. It must not kill an arbitrary pid based only on metadata.
- **GOCLI-PROTO-001.L4:** `stop` returns an acknowledgement response before the daemon shuts down its socket/runtime and removes socket artifacts.
- **GOCLI-PROTO-001.L5:** Foreground `todo daemon start [--db <path>] [--config <path>]` is the only Go CLI command allowed to exec the Clojure daemon entrypoint.
- **GOCLI-PROTO-001.L6:** In development, `todo daemon start` execs the Clojure daemon entrypoint as the long-lived foreground process, forwards `--db` and trusted daemon `--config`, streams stdin/stdout/stderr according to the parent process, and exits with the daemon process exit status.
- **GOCLI-PROTO-001.L7:** Startup/config read, compile, or runtime failures fail loudly before runtime metadata publication. The launcher must surface those failures and must not create replacement metadata or fall back to direct SQLite access.
