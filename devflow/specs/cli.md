# CLI Surface

**Document ID:** `SPEC-002` **Status:** Implemented **Last Updated:** 2026-07-23 **Related RFCs:** [RFC-019 Op-only CLI](../archive/26-07-04__op-only-cli/rfcs/2026-07-04-op-only-cli.md), [RFC-002 Task Query DSL](../rfcs/2026-06-24-task-query-dsl.md), [RFC-003 Fast JSON Socket CLI](../archive/26-06-25__go-cli-migration/rfcs/2026-06-25-fast-json-socket-cli.md), [RFC-004 Go CLI Migration](../archive/26-06-25__go-cli-migration/rfcs/2026-06-25-go-cli-migration.md) **Code:** `cli/`, `src/skein/core/weaver`, `src/skein/api/weaver/alpha.clj`, `.skein/workflows.clj`

## SPEC-002.P1 Purpose

The public CLI is two thin Go binaries with one bright line between them: `mill` owns everything that must work without a running weaver (routing/supervision, workspace bootstrap, weaver lifecycle, trusted nREPL attach), and `strand` is a pure dispatcher for everything that needs one — it resolves selection context, assembles one invoke envelope per call, and relays NDJSON back. `strand` has **zero builtin subcommands**: every command name is a weaver-registered op (SPEC-004.C63a–c). The shipped everyday command surface (`add`, `list`, `ready`, `weave`, ...) is registered by the `skein.spools.batteries` reference spool, and its per-command behavior contract lives at [`spools/batteries.md`](../../spools/batteries.md), not in this spec. Other activated reference spools register their own ops and document their per-op contracts in their spool docs.

CLI/agent list-style reads are lean and capped by default. The `list`, `ready`, and query-backed listing surfaces return an omission descriptor for attribute values above the fixed byte floor, while `show` returns the full row. Those listing surfaces also apply a default result cap of 500 before attribute assembly. If more rows match, the op fails instead of returning a partial result; the error names the total, the cap, and the remedies: narrow with `--query`/`--param`/`--state`, or pass explicit `--limit N`. Setting `--limit` above the reported total is the sanctioned full read. No pagination is part of the CLI contract; a capped-out consumer should write a narrower query. No hydration flag is part of the CLI surface; callers compose a lean listing with `show <id>` when they need a large payload. The dispatcher still relays op output verbatim as NDJSON; leanness and result caps are batteries read-surface transforms, not transport behavior.

## SPEC-002.P2 Interface

Entrypoints:

```text
strand [dispatcher-flags] <op-name> [args...]
strand --help | -h | --version
strand --dry-run [dispatcher-flags] <op-name> [args...]

mill start
mill status
mill weaver list
mill init [--workspace <dir> | --stealth]
mill weaver start [--workspace <dir>] [--name <name>] [--ready-timeout <dur>]
mill weaver status|stop [--workspace <dir>]
mill weaver repl [--stdin] [--workspace <dir>]
mill skein prime
mill strand prime
```

Dispatcher flags: `--workspace <dir>`, `--cwd <dir>`, `--worktree-root <dir>`, `--git-common-dir <dir>`, `--stdin`, `--payload name=path` (repeatable), `--timeout <dur>`, `--dry-run`, `--version`, `--help` (`-h` is an equivalent top-level alias).

## SPEC-002.P3 Dispatcher contracts

- **SPEC-002.C1:** Public `strand` commands require a reachable local `mill` entrypoint. If no mill is running or its metadata/socket is stale, commands fail non-zero with remediation to run `mill start`. If a live mill's metadata carries a different mill protocol version than the invoking `strand` binary, commands fail non-zero naming both protocol versions, both build identities when stamped, and the metadata path, with remediation to align versions (rebuild `strand` to match the running mill, or restart mill from the newer build) — never the `mill start` remedy, which is misleading while a mill runs.
- **SPEC-002.C1a:** `--workspace` selects a Skein workspace and is the highest-precedence override. Without it, implicit repo selection requires cwd to be inside a supported non-bare Git worktree; the selected workspace is the canonical repository `.skein` directory derived from the absolute Git common directory. Linked worktrees for one repository therefore select the same default workspace. Outside supported Git layouts, no-flag commands fail loudly rather than creating cwd workspaces. Runtime state, sockets, metadata, and SQLite data live under mill-owned XDG state paths, not inside the selected workspace.
- **SPEC-002.C2:** Client JSON config lives at `config.json` inside the selected workspace. Outside `mill init` bootstrap, it must exist and declare `"configFormat":"alpha"`. The alpha config format supports the low-privilege `configFormat` marker plus an optional non-blank string `"name"` declaring the workspace's portable logical weaver name; missing config, malformed config, missing/unsupported config format, unsupported keys, or wrong value types fail non-zero. `config.json` is shareable checked-in workspace config and does not store or authoritatively supply the Skein source checkout path.
- **SPEC-002.C2a:** A machine-local `config.local.json` may sit beside `config.json` as a shallow overlay of the optional keys (`"name"`; it must not carry `configFormat`). A missing overlay contributes nothing; a malformed present overlay, unknown overlay keys, or wrong value types fail non-zero. Overlay values win over `config.json`.
- **SPEC-002.C3:** Mill owns Skein source checkout resolution for launching the weaver and any thin nREPL attach client used by `mill weaver repl`. It resolves source from `SKEIN_SOURCE`, then install-time embedded source, then a canonical Skein checkout cwd containing `deps.edn`; failures are non-zero and actionable. Source is launch context, not selected workspace identity or config workspace state.
- **SPEC-002.C4:** Public strand/mill commands emit JSON machine output (`strand` responses as NDJSON; a single result is one JSON line). `--format` and config-file output format settings are not supported. All `strand` JSON output is byte-faithful — single-result stdout, stream-relay lines, and the `details=` JSON appended to stderr error messages are emitted without HTML escaping (`<`, `>`, `&` print literally, never `<`).
- **SPEC-002.C4a:** Public CLI op result objects accrete under their published op name: fields may be added, but existing fields are never removed, renamed, retyped, or repurposed. Consumers must ignore unknown fields. An op's `:returns` declaration describes the current producer shape; it does not narrow future results to exactly those fields.
- **SPEC-002.C30:** `strand [flags] <op-name> [args...]` stops dispatcher flag parsing at the first non-flag token; the op name resolves against the weaver op registry and everything after it ships verbatim as string argv. The dispatcher parses no per-command flags and interprets no argv values.
- **SPEC-002.C31:** Selection/context precedence: `--workspace` wins outright; otherwise explicit `--worktree-root`/`--git-common-dir` win over derivation; derivation runs from the effective cwd (`--cwd` when given, else process cwd) and fills only unset fields. Failed git derivation with nothing pinned fails loudly with remediation.
- **SPEC-002.C32:** Named payloads: `--stdin` reads stdin verbatim to EOF into payload slot `stdin` (opt-in only; the bin never auto-consumes a pipe); `--payload name=path` reads the named file client-side into a named slot. Duplicate slot names fail loudly before transport. The weaver never reads client-named paths.
- **SPEC-002.C33:** `--timeout <dur>` parses Go-duration style and rides in the envelope as milliseconds so the weaver can cancel or refuse; the invoked leaf's deadline class supplies the default for single-result responses (SPEC-004.C26b). Stream-class responses are never server-deadline-bounded; cancellation is client interrupt, which closes the connection and aborts the emitting handler.
- **SPEC-002.C34:** Bin-only flags work without any mill or weaver: `--version` prints one JSON object with exactly `bin_version` and `protocol_version`; `--help` / `-h` (and bare `strand`) prints static usage covering dispatcher flags, the envelope contract, the live `strand help` pointer, and `mill start` remediation; `--dry-run` prints the assembled request frame instead of sending — the standard frame shape (SPEC-002.C35) with `operation` `"invoke"`, the envelope as `arguments`, empty `options`, and the placeholder `"<dry-run>"` for both `request_id` and `weaver_id`.
- **SPEC-002.C35:** The socket request frame retains protocol version, request id, and weaver id (SPEC-004.C23); the invoke envelope rides as the operation arguments: `{name, argv, payloads, cwd, worktree_root, git_common_dir, workspace, timeout?, client{pid, version}}`. Automatic fields (frame identity, client pid/version) are not flag-settable.
- **SPEC-002.C36:** Response relay is frame-driven: a single-result response prints the result as one JSON line; a stream response (header frame, emitted NDJSON lines, terminator frame) relays each emitted line as received with per-line flush and exits by the terminator's success flag. Exit codes are contract: `0` on success, non-zero on any error, and `130` on client interrupt — including interrupt during a stream, which exits cleanly after closing the connection.
- **SPEC-002.C37:** Payload references (`:stdin` / `:payload/<name>` as whole argv values) are resolved weaver-side by the blessed parser (`skein.api.cli.alpha`, SPEC-003), never by the bin.
- **SPEC-002.C38:** Unknown op names fail non-zero with the weaver's not-found domain error carrying available op names. Malformed dispatcher flags, duplicate payload slots, no reachable mill, no running weaver, stale metadata/socket state, transport/identity failures, malformed weaver responses, database/domain errors, op arg-spec parse failures, and lifecycle hook rejections fail non-zero. Hook failures use the `hook/failed` domain error code with structured details.
- **SPEC-002.C39:** Live discovery is the core-registered `help` op, expressed as one declared, versioned schema, uniformly projected — renderings are transforms over it, and no op hand-authors usage or dispatch output. `strand help <op>` returns one response envelope, never a bare node, with stable top-level `{schema-version, operation, source, glossary, node}` (shape in SPEC-002.C44; runtime machinery in SPEC-004.C106–C112). `schema-version` versions the help-schema contract itself, independent of `protocol_version`; additive accretion does not bump it and a breaking reshape does. The node is fractal and self-similar. `strand help <op> <verb> [<verb> ...]` slices to any declared node at arbitrary depth and returns the same shape under `node`. This feature's node/operation reshape bumps `schema-version` once. The node's `returns` carries the JSON-safe return-shape explanation. `strand help` with no op returns versioned `{schema-version, ops[]}`. Two builtin meta-verbs join `help`: `strand about <op>` and `strand prime <op>`. Projection, rendering, source resolution, and the `--help` rewrite are weaver-side; the dispatcher relays verbatim argv and returned bytes without interpreting declarations.
- **SPEC-002.C44 (fractal node + versioned catalog):** The help envelope's `node` is normalized and self-similar; every key is always present with defined empty/null semantics so one recursive renderer needs no per-level branches: `name` (string), `doc` (string, `""` when undeclared), `invocation` (`{mode, flags[], positionals[]}` where `mode` is `"declared"` or `"raw-envelope"`), `returns` (return-shape explain or `null`), `hook-class` and `deadline-class` (class-name strings on invocable leaves; `null` on interior nodes and subcommand-op roots), `use-when`/`notes` (string arrays, `[]` when none), `failure-modes` (array of glossary outcome names only, `[]` when none), and `children` (node array of the same shape, `[]` for leaves). `invocation.mode` distinguishes raw-envelope from declared invocation. `operation` carries only `{name, provenance, stream?, raw-envelope}`; `source` remains the op-wide handler pointer and `glossary` the referenced-outcome closure. Catalog summary nodes follow the same leaf rule: classes are populated only when the summary node is itself a leaf. Per-entry catalog envelopes still omit `schema-version` and `glossary`.
- **SPEC-002.C45 (default rendering, `--json` floor, verbatim relay):** With no registered default help transform, `strand help` output IS the raw canonical envelope JSON. A registered default help transform (SPEC-004.C106) receives the **full envelope** and renders help instead; its output is relayed **verbatim** — the weaver's single-result success frame carries an optional `verbatim` boolean (present/`true` only for a transform-rendered `help` result) and the `strand` client then writes the raw string bytes (single trailing newline if absent) rather than the default byte-faithful JSON relay (SPEC-002.C4/C36); absent (the default) means the ordinary JSON relay, a backward-compatible frame extension. `--json` is the **sole**, **leading-only** opt-out back to the raw envelope and the raw floor when no transform is registered; it always bypasses, so a failing transform never bricks help (the failure surfaces loudly, SPEC-004.C106). The per-invocation `-t/--transform` selector is **not shipped**; other flags on the help surface are invalid (`strand help ... --foo` fails). `about`/`prime` output (JSON objects) is never transformed and never verbatim.
- **SPEC-002.C46 (`--help` grammar + `about`/`prime` meta-verbs):** After an op name, `--help`/`-h` is sugar the weaver rewrites to the `help` op, valid **only** as the final token with no other flags or attached payloads; any other shape (`strand agent --foo --help`, a non-final `--help`) is a concise redirect error pointing at `strand help ...`. This trailing-`--help` rewrite applies to **all** ops (flat, subcommand, raw-envelope) and supersedes the retired subcommand-only sole-token `<op> help|about|prime` sugar, which is migration-only and retired in alpha (TEN-000@1) — a bare `help`/`about`/`prime` word in verb position fails with the loud redirect (exemption for ops still declaring a real subcommand by that name: SPEC-004.C108). Two builtin meta-verbs join `help`: `strand about <op>` returns `{"about": <prose>, "source": <source>}` and `strand prime <op>` returns `{"prime": <prose>, "source": <source>}` — each a JSON object (never a bare string) with the op-wide `source`. Both are **arity-1** (op-level): a verb path (`strand about agent delegate`) fails loudly and redirects to `help`. Absence of declared `about`/`prime` prose returns a loud `discovery/unavailable` outcome (TEN-003), never empty success.
- **SPEC-002.C47 (dispatcher pre-op `--help` grammar):** `strand --help` / `-h` / bare `strand` with **no op** stays dispatcher usage printed by the Go dispatcher (SPEC-002.C34/C15). Once an op is named (arity ≥ 1), `--help` **must trail**: `strand --help <op>` is now an **error** (was static usage — the dispatcher set the help bit on any pre-op `--help` and ignored trailing tokens). The dispatcher distinguishes no-op `--help` (usage) from op-present pre-op `--help` (error redirecting to `strand help <op>` or a trailing `--help`), and still ships verbatim argv and interprets no arg-spec (SPEC-002.C30, TEN-006).
- **SPEC-002.C39a:** The repo coordination workspace's `land` op enriches every
  checkpoint view in the `:ready` collection returned by `start`, `next`,
  `complete`, `choose`, and `status` with `:choice-details` beside `:choices`.
  The value is the canonical string-keyed `workflow/choice-details` map for
  that materialized step, including its input declarations and route. Each
  checkpoint is joined by its own step id; non-checkpoint views omit the field.
  The shared workflow step-view remains unchanged.

## SPEC-002.P4 Mill contracts

- **SPEC-002.C14a:** `mill init` is the selected workspace bootstrap command only. It creates or completes missing alpha workspace files/directories and never contacts a weaver, initializes SQLite storage, or runs `git init`. Without `--workspace`, it creates or completes `.skein` at the canonical Git repository root and fails loudly outside supported Git layouts. With explicit `--workspace`, it bootstraps that directory directly as an isolated workspace. Bootstrap creates only missing files: selected workspace, `config.json`, `spools/`, `spools.edn`, `init.clj`, and `.gitignore`; it never overwrites existing files. A workspace carrying legacy `libs.edn`/`libs.local.edn` fails init loudly with rename remediation (`spools.edn`/`spools.local.edn`, top-level `:libs` to `:spools`) rather than being migrated or ignored. Created config uses only `"configFormat":"alpha"` and does not persist source. Generated `spools.edn` seeds `skein.spools/batteries {:skein/source-root "spools/batteries"}`; the relative coordinate is resolved at runtime against the mill-resolved Skein source checkout and persists no absolute checkout path. Deleting that entry is the supported visible opt-out. Repo `.skein/.gitignore` ignores local overlays and accidental runtime artifacts including `config.local.json`, `init.local.clj`, `spools.local.edn`, `state/`, `data/`, `weaver.*`, and SQLite artifacts. Generated `init.clj` requires `skein.api.current.alpha` and `skein.api.runtime.alpha`, captures `(current/runtime)`, and declares batteries as a normal `:spools`-guarded `contribute`/`reconcile` module so fresh workspaces get the shipped command surface through the approved-coordinate path. Repo-world bootstrap (no `--workspace`) additionally ensures the repository-root agent guidance file carries a marker-guarded `## Skein / strand` section pointing new agents at `mill skein prime` and `mill strand prime`: it appends the section to whichever of `AGENTS.md`/`CLAUDE.md` exist (idempotent via the `<!-- mill:skein-prime -->` open marker with a matching `<!-- /mill:skein-prime -->` close marker bounding the block, append-only, never rewriting existing prose) and creates `AGENTS.md` when neither exists. Explicit `--workspace` bootstrap performs no guidance injection.
- **SPEC-002.C14b:** `mill init --stealth` is the explicit exception to C14a's repository guidance injection. It is valid only without
  `--workspace` and selects the canonical Git-root `.skein`. Before writing anything, it refuses tracked `.skein` paths and malformed
  mill-owned marker blocks. It bootstraps ordinary workspace files without editing shared `AGENTS.md` or `CLAUDE.md`, then maintains this
  exact block in `.git/info/exclude`: `# mill:skein-stealth`, `/.skein`, `/CLAUDE.local.md`, `# /mill:skein-stealth`. It maintains the
  exact C14a guidance block in an untracked `CLAUDE.local.md`; a tracked `CLAUDE.local.md` is left untouched. An absent owned file is
  created with the exact block. A non-empty marker-free file keeps its bytes and gains a separating newline plus the block. Exact blocks,
  including their final newline, are unchanged; partial, duplicate, reordered, or edited blocks are semantic refusals. The existing `config_dir` and
  `config_file` success keys gain a typed `stealth` object with closed `git_exclude` (`path`, `status`), `claude_guidance` (`path`,
  `status`), and `codex_guidance` (`status`, `suggested_text`) children. File status values are `created`, `updated`, `unchanged`, and,
  for Claude guidance only, `skipped-tracked`; Codex status is `manual-required`. Go response structs validate this shape before emission.
  Semantic refusals use code `mill/init-stealth-refused` and exactly `path`, `target`, `state`, and `remediation` details. Target values are
  `tracked-skein`, `git-exclude`, and `claude-guidance`; state values are `tracked`, `start-only`, `end-only`, `duplicate-start`,
  `duplicate-end`, `reversed`, and `edited`. Ordinary `mill init` output remains unchanged.
- **SPEC-002.C15:** `mill` uses Cobra for its command tree and help text. The `strand` dispatcher owns its fixed flag set directly and must clearly describe the dispatcher contract in `--help`; it has no command tree to document.
- **SPEC-002.C16:** `mill weaver start` resolves the selected workspace and starts that workspace's Clojure weaver as a mill child process using mill-derived XDG state/data dirs and mill-resolved source, returning selected workspace JSON status without running the weaver in the foreground. `--name <name>` sets a non-empty friendly weaver name; when omitted, mill resolves the name from `config.local.json` `"name"`, then `config.json` `"name"`, then the selected workspace basename. `--ready-timeout <dur>` sets the startup wait budget as a positive Go duration; omitted uses 5m. If source resolution fails, startup fails with remediation naming the accepted sources. The weaver owns storage selection/preparation and loads selected workspace startup files in order: `init.clj`, then `init.local.clj`; missing files are skipped and present failing files fail startup loudly. Startup waiting allows for JVM boot plus trusted config work before reporting ready status. On timeout, mill terminates the still-starting weaver rather than leaving an orphan. Weaver stdout/stderr are captured to `weaver.log` in the weaver state dir — appended across restarts, never forwarded to mill's own log, and left in place by artifact cleanup so a failed boot stays post-mortem readable; startup failures carry the log path and its tail in the error, and weaver status JSON carries `log_path`.
- **SPEC-002.C17:** `mill weaver repl` verifies the selected workspace's weaver is running and attaches to its nREPL endpoint using mill-resolved source launch context. Mill does not proxy nREPL traffic. Any launched local process is a thin transport/UI attach client only; user forms evaluate in the weaver JVM.
- **SPEC-002.C17a:** `mill skein prime` and `mill strand prime` print harness-agnostic agent orientation to stdout and need no running mill or weaver. Content ships in the Skein source checkout, not the `mill` binary: `mill <topic> prime` resolves the Skein source (SPEC-002.C3 precedence), reads the prime manifest at the frozen path `docs/prime/index.json` (`{"version": N, "topics": {<topic>: <repo-relative file>}}`), and renders the file the manifest names for the topic, so an already installed `mill` prints current orientation text from a newer checkout. Manifest topic paths use slash-form relative paths with no volume prefix and no empty, `.`, or `..` segments. Rendering interprets exactly one template token: every `{{.Source}}` occurrence becomes the resolved absolute checkout path, and no other template construct is interpreted; topic files must use nothing else. The manifest path never moves; the manifest carries a schema `version`, and a `mill` reading a version it does not support fails non-zero instructing an upgrade rather than rendering. Unresolvable source, missing manifest, unknown topic, invalid topic path, and unreadable topic files all fail non-zero with the failing path. `mill strand prime` prints the strand planning/tracking workflow. These commands are the shipped replacement for the former on-disk strand/skein agent skills.
- **SPEC-002.C18:** `mill weaver repl --stdin` sends stdin Clojure forms to the selected running weaver nREPL for direct evaluation in the weaver JVM, prints one direct normal Clojure result per top-level form, and exits non-zero on read, eval, or transport errors. It does not impose a JSON or EDN response envelope.
- **SPEC-002.C19:** `mill weaver repl` and `mill weaver repl --stdin` are the public CLI paths for users and agents that need to run trusted spool-workspace Clojure code against a running weaver workspace.
- **SPEC-002.C20:** `mill weaver status` reports health, friendly name, selected config/data paths, weaver storage metadata (`database_kind`, `database_label`, `database_path` — canonical path for `sqlite-file`, explicitly `null` for `sqlite-memory`), the weaver log path (`log_path`), pid, weaver identity, socket endpoint, and nREPL endpoint when running, sourced from mill supervision state, runtime metadata, and a minimal socket health check. `mill weaver stop` stops only the selected workspace weaver child through mill supervision (signal-driven; SPEC-004.C26a) and cleans that workspace's runtime metadata/socket; non-supervised metadata-discovered weavers are signalled by pid with the same loud staleness checks.
- **SPEC-002.C20a:** `mill weaver list` returns a JSON array for editor/tooling discovery. Rows include `config_dir`, `state_dir`, `data_dir`, `database_kind`, `database_label`, `database_path`, `name`, `log_path`, and `state`; running rows also include `pid`, `weaver_id`, `socket_path`, `nrepl`, and `started_at`. The list includes supervised children and metadata-discovered weavers under the mill state root. Malformed metadata fails loudly instead of being silently repaired.
- **SPEC-002.C20b:** No public CLI command or flag selects weaver storage kind. CLI metadata/status validation compares `database_kind` and `database_label` for all storage kinds and `database_path` only for `sqlite-file`, failing loudly on malformed or inconsistent storage metadata. Mill supervises file-backed workspace weavers only and rejects other storage kinds in mill-managed weaver metadata.
- **SPEC-002.C21:** Runtime spool workspace, runtime transformation, lifecycle hook, attribute archive/migration, and registry mutation operations happen through selected workspace `init.clj`, blessed explicit-runtime `skein.api.*.alpha` namespaces, and trusted REPL workflows, not through dispatcher flags or mill commands. There are no package/spool/hook/batch activation commands, no public command archives attributes, and no public command migrates attribute storage.

## SPEC-002.P5 Removed surface and non-goals

- **SPEC-002.C40:** The builtin strand command tree is removed with no aliases (TEN-000@1): `init`, `add`, `update`, `show`, `supersede`, `burn`, `list`, `ready`, `graph subgraph`, `weave`, `query list|explain`, `pattern list|explain`, the `op <name>` prefix, and `weaver start|repl|stop|status` as strand subcommands. Everyday command behavior is batteries ops ([`spools/batteries.md`](../../spools/batteries.md)); bootstrap/lifecycle is mill. Client-side attribute-input flags (`--attr`, `--attr-file`, `--attr-stdin`, `--attributes-stdin`, `--edge`, `--state`, `--query`, `--param`) are op argv now — the dispatcher carries payload slots instead.
- **SPEC-002.C41:** No stream-in: requests stay one bounded envelope; bounded bulk input is a payload parsed and applied op-side in one transaction. Unbounded live ingestion belongs to spools inside the weaver, never a dispatcher flag.
- **SPEC-002.C42:** No hook bypass surface of any kind — permanent invariant.
- **SPEC-002.C43:** Historical per-command contracts (old SPEC-002.C5–C13c, C22–C25) are superseded; their behavior equivalences are recorded in `spools/batteries.md` and the archived op-only-cli feature folder.

## SPEC-002.P6 Deferred

`by-attr`, bespoke dependency inspection commands, `link`, `done`, `batch`, arbitrary JSON graph patch commands, public CLI EDN query expressions, query registry mutation/authoring commands, `--query-file`, pattern registry mutation commands, plugin/package commands, compatibility `todo` binaries, legacy Clojure CLI entrypoints, and fallback discovery of old `atom` workspaces or `daemon.*` artifacts are not part of the public CLI.
